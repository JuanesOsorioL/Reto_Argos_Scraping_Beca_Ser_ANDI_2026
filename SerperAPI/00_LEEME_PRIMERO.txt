⚡ INICIO RÁPIDO (5 MINUTOS)

1. Obtén API key: https://serper.dev/ (gratis)

2. Copia en .env:
   SERPER_API_KEY=tu_key_aqui

3. Instala dependencias:
   pip install -r requirements.txt

4. Inicia servidor:
   python api_runner.py

5. En otra terminal, prueba:
   curl -X POST http://localhost:8004/scrape/serper/prueba

6. Ve el progreso:
   curl http://localhost:8004/progress

🔑 CONCEPTOS CLAVE

Rate Limit (429):
  Si intentas más de 100 requests/minuto en Serper
  → Sistema detecta automáticamente
  → Pausa inteligentemente
  → Espera 15 minutos
  → Reintenta automáticamente
  → NO pierdes datos

Reanudación:
  Si se cae la app
  → Todos los datos ya están en PostgreSQL
  → Archivo serper_progress.json guarda dónde quedó
  → Llamar POST /resume
  → Continúa exactamente donde paró
  → Salta queries ya procesadas

Webhooks:
  Cuando scraping termina
  → Envía POST a n8n
  → n8n recibe evento
  → n8n puede notificar (Telegram, Slack, etc)
  → n8n puede disparar más acciones

🚀 EJEMPLO FLUJO PRODUCCIÓN

1. User: curl -X POST http://localhost:8004/scrape/serper
                         ↓
2. API retorna run_id e inicio (no bloquea)
                         ↓
3. Background: main.py empieza loop
   - Para cada query:
     a) Verificar si ya procesada (salta)
     b) Llamar Serper
     c) Si 429: esperar + reintentar
     d) Enriquecer (visitando URLs)
     e) Insertar en PostgreSQL
     f) Guardar progreso
                         ↓
4. Mientras tanto: n8n hace polling GET /progress
   - Ve "corriendo", "pausado_sin_tokens", "ok", "error"
   - Si "pausado_sin_tokens": espera + resume automáticamente
   - Si "ok": notifica Telegram "✅ Completado: 3.450 registros"
                         ↓
5. Datos en PostgreSQL: nunca se pierden
   Progreso guardado: puede reanudar en cualquier momento
   Notificación enviada: n8n supo que terminó
