🔑 CARACTERÍSTICAS CLAVE
════════════════════════════════════════════════════════════════════════════════

1. DETECCIÓN AUTOMÁTICA DE 403
   ├─ Foursquare devuelve 403 cuando alcanzas 5.000 requests/hora
   ├─ El sistema lo detecta automáticamente
   ├─ Lee header X-RateLimit-Reset (cuándo se resetea)
   ├─ Espera el tiempo correcto
   ├─ Reintenta automáticamente
   └─ Continúa desde donde quedó

2. REANUDACIÓN SIN PERDER DATOS
   ├─ Guarda estado en foursquare_progress.json
   ├─ Guarda estado en PostgreSQL
   ├─ Si se cacha → reinicia y continúa
   └─ NUNCA pierdes datos

3. CONTROL REMOTO HTTP
   ├─ POST /scrape/foursquare → Iniciar
   ├─ GET /status → Ver estado
   ├─ GET /progress → Ver progreso
   ├─ POST /reset → Limpiar archivos
   └─ GET /docs → Documentación interactiva

4. NOTIFICACIONES A N8N
   ├─ Webhook cuando se pausa por 403
   ├─ Webhook cuando completa
   ├─ n8n puede notificar a Telegram
   ├─ n8n puede reanudar automáticamente
   └─ n8n puede hacer muchas cosas más

⚡ SISTEMA DE PAUSA AUTOMÁTICA (lo más importante)
════════════════════════════════════════════════════════════════════════════════

Flujo automático cuando se detecta 403:

Foursquare API devuelve 403 (rate limit)
        ↓
scraper.py lo detecta
        ↓
Lanza RateLimitException
        ↓
main.py lo captura automáticamente
        ↓
Guarda estado (BD + JSON)
        ↓
Envía webhook a n8n (notifica)
        ↓
Espera 3600 segundos (1 hora, configurable)
        ↓
Reintenta automáticamente
        ↓
Continúa desde combinación (keyword, ciudad) donde quedó
        ↓
✅ NUNCA PIERDES DATOS
