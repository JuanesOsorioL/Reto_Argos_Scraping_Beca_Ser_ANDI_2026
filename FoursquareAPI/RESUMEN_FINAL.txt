🎯 CARACTERÍSTICAS PRINCIPALES
════════════════════════════════════════════════════════════════════════════════

1️⃣ DETECCIÓN AUTOMÁTICA DE 403 (RATE LIMIT)

   Flujo automático:
   ├─ Foursquare devuelve 403
   ├─ scraper.py lo detecta
   ├─ Lee header X-RateLimit-Reset
   ├─ Lanza RateLimitException
   ├─ main.py lo captura
   ├─ Guarda progreso (BD + JSON)
   ├─ Envía webhook a n8n
   ├─ Espera RATE_LIMIT_SLEEP_SECONDS (3600 segundos = 1 hora)
   ├─ Reintenta automáticamente
   └─ Continúa desde donde quedó

   Límites Foursquare:
   ├─ 5.000 requests/hora (gratis)
   ├─ 500 a otros endpoints/hora
   ├─ Status: 403 cuando se alcanza
   ├─ Header X-RateLimit-Reset: UNIX timestamp del reset
   └─ Nuestro sistema: espera y reintenta automáticamente

2️⃣ REANUDACIÓN INTELIGENTE

   Si falla o se apaga:
   ├─ Guarda estado en foursquare_progress.json
   ├─ Guarda estado en PostgreSQL tabla foursquare_progress
   ├─ Reinicia → carga estado
   ├─ Continúa desde combinación (keyword, ciudad) donde quedó
   └─ NUNCA pierdes datos

3️⃣ GUARDADO DE DATOS (MÚLTIPLES FORMATOS)

   PostgreSQL (OBLIGATORIO):
   ├─ Tabla: raw.foursquare_ferreterias
   ├─ Registros: 1,500+
   ├─ Columnas: 45+ (Argos + extras Foursquare)
   ├─ Índices: 6 (para búsquedas rápidas)
   └─ ON CONFLICT: evita duplicados automáticamente

   JSON Files (CONFIGURABLE):
   ├─ foursquare_progress.json       — Estado actual
   ├─ foursquare_ferreterias.jsonl   — Respaldo línea a línea (SIEMPRE)
   ├─ output/foursquare_raw_responses.json — Respuestas crudas
   └─ output/foursquare_flat_results.json  — Normalizadas

   Excel (OPTIONAL):
   └─ foursquare_ferreterias.xlsx — Datos listos para analizar

4️⃣ CONTROL REMOTO VÍA HTTP (8 ENDPOINTS)

   POST /scrape/foursquare  → Iniciar scraping
   GET  /status             → Ver estado actual
   GET  /progress           → Ver progreso detallado
   POST /pause              → Pausar (manual)
   POST /resume             → Reanudar (manual)
   POST /reset              → Limpiar archivos (NO BD)
   GET  /stats              → Estadísticas de BD
   GET  /health             → Health check (Docker)

   Respuesta tipo:
   {
     "status": "completado",
     "run_id": "abc-123-def...",
     "total_insertadas": 1500,
     "total_aprobadas": 1200,
     "rate_limit_count": 1
   }

5️⃣ WEBHOOKS A N8N (NOTIFICACIONES AUTOMÁTICAS)

   Eventos:
   ├─ WEBHOOK_ON_PAUSE    → Cuando se pausa por 403
   ├─ WEBHOOK_ON_COMPLETE → Cuando termina
   └─ WEBHOOK_ON_ERROR    → Cuando hay error fatal

   n8n puede:
   ├─ Notificar a Telegram (mensaje)

   ├─ Guardar a Google Sheets (estadísticas)

════════════════════════════════════════════════════════════════════════════════
🆘 SOPORTE RÁPIDO
════════════════════════════════════════════════════════════════════════════════

Problema: "403 Rate Limit detectado"
Solución: NORMAL. El sistema pausa automáticamente y reintenta después de 1 hora.

Problema: "401 Unauthorized"
Solución: API Key incorrecta. Obtén nueva en https://developer.foursquare.com

Problema: "Connection refused to PostgreSQL"
Solución: PostgreSQL no está corriendo. Inicia con: docker-compose up -d postgres

Problema: "Port 8006 already in use"
Solución: Cambiar API_PORT en .env (ej: 8007)

Problema: "Webhook not working"
Solución: n8n no está en http://localhost:5678. Actualizar WEBHOOK_ON_* en .env
